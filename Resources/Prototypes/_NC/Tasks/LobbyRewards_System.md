# Task: Система наград и экономики в лобби (Night Coins & Meta-Inventory)

## 1. Цель
Создать функциональную панель в лобби игры, где игроки могут просматривать свой баланс валюты (Night Coins) и список доступных предметов-наград. Система должна позволять выбирать предметы для "выноса" в следующий раунд.

## 2. Дизайн и Стилистика (Cyberpunk Aesthetic)
Интерфейс должен соответствовать духу киберпанка:
- **Цветовая палитра**: Глубокий черный/темно-серый фон (`#0d0d0d`), неоновые акценты (Cyan `#00ffff` для обычных элементов, Magenta `#ff00ff` для редких/важных).
- **Визуальные элементы**: Тонкие рамки, использование сетки (grid), эффекты "свечения" (через StyleBox) и строгие геометрические формы.
- **Шрифты**: Моноширинные или футуристичные (из доступных в Robust).

## 3. Функциональные требования

### 3.1. Панель "Склад" (Meta-Vault)
- Отображение баланса **Night Coins**.
- Список предметов в мета-инвентаре (хранится в БД между раундами).
- Информация о предмете: название, описание, статус (доступен/выбран).
- Кнопка **"Подготовить к высадке"**: помечает предмет для спавна при старте раунда.

### 3.2. Экономика и Награды
- **Night Coins**: Валюта, начисляемая за выполнение целей раунда (Antag objectives, Job goals).
- **Выдача предметов**: Предметы могут выдаваться администрацией или как автоматическая награда за достижения.
- **Предметы**: Предметы, выбранные для раунда, НЕ удаляются из мета-инвентаря после старта.

## 4. Техническая архитектура

### 4.1. База данных (Persistence)
Необходимо расширить схему БД:
- Таблица `nc_player_economy`: `user_id`, `night_coins_balance`.
- Таблица `nc_meta_inventory`: `id`, `user_id`, `item_prototype_id`, `metadata`.

### 4.2. Серверная часть (Content.Server)
- `LobbyRewardsSystem`: Обработка запросов от клиента, связь с БД.
- `RewardSpawnSystem`: Логика спавна выбранных предметов в инвентарь игрока при входе в игру (`PlayerSpawnCompleteEvent`).
- **Валидация**: Проверка наличия предмета и достаточного баланса перед любой операцией.

### 4.3. Клиентская часть (Content.Client)
- Обновление `LobbyGui`: Добавление новой вкладки или кнопки для открытия панели наград.
- `LobbyRewardsEui`: Использование Entity User Interface для синхронизации данных о наградах между сервером и клиентом.

## 5. План реализации (Этапы)
1. **Этап 1**: Создание архитектуры БД и базовых CRUD-операций для валюты и предметов.
2. **Этап 2**: Реализация серверной логики подготовки предметов к раунду.
3. **Этап 3**: Разработка UI в лобби с использованием киберпанк-стилистики.
4. **Этап 4**: Интеграция спавна предметов при старте раунда.
5. **Этап 5**: Тестирование и валидация безопасности.
